## Common Operation Workflows

### Add Subscription Workflow 
1. **Input Parsing Phase**
   - **Extract season number from user input first**: 
     - If input contains number suffix (e.g., "不良人6", "进击的巨人4"), extract the number as season
     - "不良人6" → main title: "不良人", season: 6
     - "进击的巨人4" → main title: "进击的巨人", season: 4
   - **Handle explicit season expressions**:
     - "xxx第X季" / "xxx Season X" / "xxx 最新季" → extract accordingly
   
2. **Media Search Phase (search_media)**
   - Use the **extracted main title** (without number suffix) for search
   - Use the search_media tool **without using season parameter**
   - Example: User inputs "不良人6" → search for "不良人"
   
3. **Main Series Identification from Search Results**
   - **Critical Rule**: The main series is the entry with the **shortest/base title** that matches the search term
   - **Exclude entries that contain season/year indicators in their title**:
     - ❌ "画江湖之不良人6 (2023)" - contains season number in title
     - ❌ "xxx 第X季" - contains explicit season marker
     - ❌ "xxx Season X" - contains explicit season marker
     - ✅ "画江湖之不良人" - this is the main series
   - **Selection Logic**:
     - From results matching the search term, select the one WITHOUT numeric season suffix
     - If multiple candidates remain, prefer the one with the earliest first_air_date/release_date
   - Obtain the tmdbid of the identified main series
   
4. **Duplicate Main Series Handling (query_media_detail)**
   - **When multiple main series entries share the same base title:**
     - Use query_media_detail to query each series
     - Check the `type` field in each response to identify media type
     - **Default Priority**: Select the entry where `type` contains "Animation" / "动画"
   
5. **Season Validation Phase (query_media_detail)**
   - Use query_media_detail with the **main series tmdbid** to get complete media information
   - Verify the extracted season number exists in `season_info` array
   - If the requested season doesn't exist, inform the user of available seasons
   
6. **Add Subscription Phase (add_subscribe.py)**
   - Pass parameters: tmdbid (main series), year, season (from step 1), mediatype.
   - Apply default subscription rules
   
7. **Result Reporting**
